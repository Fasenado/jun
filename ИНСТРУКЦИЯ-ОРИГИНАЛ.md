# Как получить оригинальный репозиторий без Git

## Вариант 1: Автоматический скрипт

Запустите в PowerShell (от имени администратора или обычного):

```powershell
cd "c:\Users\Falon\Desktop\next.junni.co.jp-master"
.\clone-original.ps1
```

Скрипт создаст папку `next.junni.co.jp-original` на рабочем столе с чистым клоном.

---

## Вариант 2: Ручная загрузка

1. **Скачайте ZIP вручную:**
   - https://github.com/junni-inc/next.junni.co.jp/archive/refs/heads/master.zip
   - Сохраните на рабочий стол

2. **Распакуйте** в папку `next.junni.co.jp-original`

3. **Запустите** `setup-submodules.ps1` из корня распакованной папки

4. **Выполните:**
   ```bash
   npm install
   npm run dev
   ```

---

## Вариант 3: Установите Git (рекомендуется)

С Git всё проще:

```bash
git clone https://github.com/junni-inc/next.junni.co.jp.git
cd next.junni.co.jp
git submodule update --init --recursive
npm install
npm run dev
```

---

## Скрипты используют точные коммиты

`setup-submodules.ps1` и `clone-original.ps1` загружают подмодули с фиксированными коммитами (2022-11):

- ore-three: `b7615d8`
- power-mesh: `7785128`

## Настройка визуала

Для тонкой настройки яркости и bloom отредактируйте `src/ts/config/postProcess.ts`:

- `exposure` — экспозиция (0.8–1.0)
- `bloomThreshold` — порог bloom (0.5–0.7)
- `gamma` — гамма-коррекция (обычно 1/2.2)

## Возможные фиксы после клонирования

Репозиторий использует подмодули `ore-three` и `power-mesh`, API которых мог измениться. Если появятся ошибки:

- `isAnimatingVariable` → заменить на `isAnimating`
- `ore-three-ts` → заменить импорты на `ore-three`
- PowerMesh/PowerReflectMesh — возможны отличия в конструкторе

Эти правки уже были внесены в вашу текущую копию; в свежем клоне их придётся сделать заново.
